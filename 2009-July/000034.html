Hi All,

I'm hoping this listserv still has some subscribers.  We recently moved 
an old SGI workhorse server across networks. For whatever reason, dx is 
the only application that refuses to automatically forward the 
connection. 'ssh -X [host]' used to do the trick, and still does for 
every other X11 application. 'dx,' however, throws this error (or 
something similar, depending on the OS/X11 client):

[jpierce@[host] bin_sgi]$ dx
X11 connection rejected because of wrong authentication.
X connection to localhost:23.0 broken (explicit kill or server shutdown).

In order to get it working, one has to tunnel the X11 connection via the 
DISPLAY environment variable. This is of limited success, though, as 
some of our users aren't particularly *nix savvy, and others are 
experiencing firewall issues on their external networks.

Has anybody seen this behavior before? I've already checked for 
references to the old IP in the usual spots.

Thank you very much,
