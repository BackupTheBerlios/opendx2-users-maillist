Hello opendx users,
Since upgrade to 4.4.4 version I have noticed something probably a bug 
but I am not sure.
Opening a visual program with control panel aspects set to dialogue 
style and then
saving this visual program with DX will reset the control panels to a 
developer style  ...
... doing nothing else than saving the program.
I have checked both files '*.net'/'*.cfg' before and after saving the 
only difference appears at the line
describing the control panel ex:
// panel[1]: position = (0.0958,0.0638), size = 0.2589x0.8505, startup = 
1, devstyle = 0

After saving with DX the file will be:
// panel[1]: position = (0.0958,0.0638), size = 0.2589x0.8505, startup = 
1, devstyle = 0, screen = 0

where you can see that a 'screen = 0' instruction now appears as last 
argument of the command.
When deleting this 'screen = 0' instruction the control panels then get 
back to the dialogue style.

An idea about this ?
Shall I report this as an OpenDX bug ?

Sébastien
